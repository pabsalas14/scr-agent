¿Qué es Centinela IA?

Centinela IA es una plataforma empresarial de auditoría automática de código que utiliza Inteligencia Artificial y análisis forense para detectar vulnerabilidades, patrones de seguridad y gestionar el ciclo completo de remediación. Diseñada específicamente para instituciones financieras y empresas que requieren cumplimiento normativo riguroso (PCI-DSS, CNBV, OWASP, CWE, CVSS).

Inteligencia: Agentes Especializados

Tres agentes cognitivos trabajan en paralelo para análisis integral de riesgos bancarios

🔍

Inspector Principal

Orquestación y diagnóstico inicial. Mapea la arquitectura, identifica puntos de riesgo críticos y coordina análisis paralelos de los otros agentes.

⚖️

Fiscal de Análisis

Clasificación y puntuación de riesgos. Calcula CVSS, mapea a OWASP/CWE, valida cumplimiento normativo y genera reportes de impacto financiero.

🔬

Detective Forense

Investigación profunda y anomalías. Analiza timeline de commits, detecta patrones sospechosos, identifica logic bombs y rastrea intención maliciosa.

📌 Demo Interactiva: Credit Score Engine

Ejemplo real de detección de vulnerabilidad crítica en sistema de scoring financiero

🚨 Vulnerabilidad Detectada: Logic Bomb

Severidad: CRITICAL | Tipo: Código Malicioso | Impacto: Fraude Financiero

Código Vulnerable:

def adjust_score(user_id, score):
    # LOGIC BOMB: Ajuste manual para usuario específico
    if user_id == 31377:  # ⚠️ Usuario hardcodeado
        score = 850  # Fuerza máximo score sin auditoría ❌

    # Falta logging de esta operación crítica
    return score

Diagnóstico (Agentes):

🔍 Inspector: Detectó usuario hardcodeado en lógica de scoring

⚖️ Fiscal: CVSS 9.8 CRITICAL - Fraude potencial de miles USD por transacción

🔬 Detective: Código inyectado en último commit de usuario desconocido (no en equipo)

Código Remediado:

def adjust_score(user_id, score):
    # ✅ Validación y auditoría completa
    if not is_authorized_scorer(user_id):
        raise UnauthorizedAccessError(f"Usuario {user_id} no autorizado")

    # Registrar cada cambio
    audit_log.info(f"Score adjustment: user={user_id}, before={score}")

    # Aplicar ajuste solo si cumple políticas
    if is_policy_compliant(score):
        return score
    else:
        raise PolicyViolationError("Score no cumple políticas")

    audit_log.info(f"Score adjustment: user={user_id}, after={score}")

✅ Resultado: Riesgo mitigado | Auditoría completa | Sistema asegurado

⏱️ MTTR: 2.3 horas (detección + remediación + validación)

Desventajas y Limitaciones

🎯 Contexto Requerido

  • Requiere configuración inicial de reglas y políticas
  • La calidad depende de parámetros precisos
  • Necesita ajustes según tecnología específica
  • Falsos positivos requieren tuning iterativo

⚠️ Limitaciones de IA

  • No detecta lógica comprometida deliberadamente (sin patrones)
  • Limitado a patrones conocidos (0-day vulnerabilities)
  • Necesita contexto arquitectónico para análisis profundo
  • Puede generar falsos positivos en código especial

🎯 Curva de Aprendizaje

  • Equipo necesita capacitación en plataforma
  • Integración con workflows toma tiempo inicial
  • Primeras auditorías generan muchos hallazgos
  • Requiere gestión de expectativas en rollout

⚠️ Limitaciones Técnicas

  • Requiere acceso a código fuente completo
  • Análisis estático tiene limitaciones vs dinámico
  • Rendimiento decrece en repos gigantes (>5M LOC)
  • Algunas vulnerabilidades requieren pruebas manuales

🔄 Cómo Mitigamos Estas Limitaciones

  • Agentes Múltiples: Combinamos 3 enfoques (Inspector, Fiscal, Detective) para máxima precisión
  • Validación Humana: Integramos workflows de aprobación y revisión por expertos
  • Tuning Automático: Sistema aprende de falsos positivos con feedback loop
  • Documentación Completa: Guías de implementación, mejores prácticas y troubleshooting
  • Soporte Premium: Equipo experto para custom policies y casos especiales
  • Análisis Híbrido: Combinamos estático + dinámico + comportamental para cobertura total

Arquitectura Técnica

🏗️ Stack Tecnológico

  • Frontend: React 19 + TypeScript + Tailwind CSS + Vite (SPA moderna)
  • Backend: Node.js + Express + Prisma ORM (microservicios listos)
  • Base de Datos: PostgreSQL con caché Redis (HA ready)
  • Análisis: AST parsing + ML models + Node.js workers (paralelo)
  • Real-time: WebSockets (Socket.io) para notificaciones instantáneas
  • Queue: Bull Queue para análisis asincrónico y escalable

  • Deployment: Docker + Docker Compose + K8s ready
