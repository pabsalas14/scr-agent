¿Cómo se Determina la Criticidad?

Sistema de clasificación basado en CVSS y análisis contextual

📊 CVSS Score (Common Vulnerability Scoring System)

Es la métrica base que evalúa la vulnerabilidad de forma técnica:

9.0 - 10.0
🔴 CRÍTICO
Explotable remotamente, sin autenticación, impacto máximo
7.0 - 8.9
🟠 ALTO
Impacto significativo, fácil de explotar, requiere poco conocimiento
4.0 - 6.9
🟡 MEDIO
Impacto moderado, requiere ciertos requisitos previos
0.1 - 3.9
🟢 BAJO
Impacto mínimo, difícil de explotar, o requiere permisos

🔍 Factores de CVSS Considerados

Vector de Acceso

Remoto: Máximo riesgo
Local: Riesgo menor
Física: Riesgo mínimo

Complejidad de Ataque

Baja: Se explota fácilmente
Alta: Requiere bypass de seguridad

Privilegios Requeridos

Ninguno: Máximo riesgo
Usuario: Riesgo medio
Admin: Riesgo bajo

Impacto de Confidencialidad

Alto: Datos sensibles expuestos
Parcial: Algunos datos afectados
Ninguno: Sin exposición

Impacto de Integridad

Alto: Modificación sin restricción
Parcial: Cambios limitados
Ninguno: No afectada

Impacto de Disponibilidad

Alto: Servicio no disponible
Parcial: Degradación de servicio
Ninguno: No afectada

📋 Ejemplos de Clasificación

🔴 CRÍTICO: SQL Injection en Login

CVSS: 9.8 | Contexto: App web pública | Datos: Bases de datos de usuarios

Por qué: Fácil de explotar (acceso remoto, sin autenticación), impacto máximo (acceso a BD completa)

🟠 ALTO: XSS en Formulario

CVSS: 6.1 | Contexto: Datos de usuario | Datos: Sesiones, cookies

Por qué: Se puede explotar remotamente pero requiere que el usuario interactúe

🟡 MEDIO: Configuración Débil en Staging

CVSS: 5.3 | Contexto: Servidor no producción | Datos: Datos de prueba

Por qué: Impacto moderado, pero es entorno de staging, no crítico

🟢 BAJO: Comentario TODO con Contraseña

CVSS: 3.1 | Contexto: Código fuente privado | Datos: Cuenta de prueba

Por qué: Requiere acceso al código fuente (no remoto), contraseña es de prueba no producción

Ejemplos & Casos de Uso

Vulnerabilidades reales detectadas y remediadas

01

SQL Injection

Detectado en: auth/login.js:45

❌ Código Vulnerable: 
const query = `SELECT * FROM users WHERE email = '${email}'`;
✅ Solución: 
const query = 'SELECT * FROM users WHERE email = ?';
db.query(query, [email]);

Severidad: CRÍTICO

CVSS Score: 9.8 (Crítico)

02

XSS (Cross-Site Scripting)

Detectado en: components/UserProfile.jsx:78

❌ Código Vulnerable: 
<div dangerouslySetInnerHTML={{__html: userInput}} />
✅ Solución: 
<div>{userInput}</div>

Severidad: ALTO

CVSS Score: 6.1 (Medio)

03

Credenciales en .env

Detectado en: config/.env:12

❌ Código Vulnerable: 
DATABASE_URL=postgres://user:password123@db.com
✅ Solución: 
// .env.example (committed)
DATABASE_URL=postgres://user:PASSWORD@db.com

// Add to .gitignore
.env
.env.*.local

Severidad: ALTO

Impacto: Exposición de credenciales de BD

04

Command Injection

Detectado en: utils/shell.js:34

❌ Código Vulnerable: 
const result = exec(`ls -la ${userInput}`);
✅ Solución: 
const { execFile } = require('child_process');
execFile('ls', ['-la', userInput], (err, stdout) => {...});

Severidad: CRÍTICO

CVSS Score: 9.6 (Crítico)

05

Path Traversal

Detectado en: routes/files.js:56

❌ Código Vulnerable: 
fs.readFile(`/uploads/${req.query.file}`, (err, data) => {...});
✅ Solución: 
const path = require('path');
const uploadDir = path.resolve('/uploads');
const filePath = path.resolve(uploadDir, req.query.file);
if (!filePath.startsWith(uploadDir)) throw new Error('Access denied');

Severidad: ALTO

CVSS Score: 7.5

06

CORS Misconfiguration

Detectado en: middleware/cors.js:8

❌ Código Vulnerable: 
app.use(cors({ origin: '*', credentials: true }));
✅ Solución: 
app.use(cors({
  origin: ['https://example.com', 'https://app.example.com'],
  credentials: true,
  methods: ['GET', 'POST']
}));

Severidad: ALTO

CVSS Score: 6.8

07

Weak Cryptography

Detectado en: auth/crypto.js:23

❌ Código Vulnerable: 
const hash = md5(password); // MD5 es quebrantable
✅ Solución: 
const bcrypt = require('bcrypt');
const hash = await bcrypt.hash(password, 10);

Severidad: MEDIO

CVSS Score: 5.9

08

Insecure Deserialization

Detectado en: cache/redis.js:45

❌ Código Vulnerable: 
const obj = JSON.parse(untrustedData);
// No valida estructura
✅ Solución: 
const schema = Joi.object({
  id: Joi.number().required(),
  email: Joi.string().email().required()
});
const { error, value } = schema.validate(JSON.parse(untrustedData));

Severidad: ALTO

CVSS Score: 8.1

09

Missing Rate Limiting

Detectado en: routes/api.js:12

❌ Código Vulnerable: 
app.post('/login', (req, res) => {
  // Sin límite de intentos
  validateUser(req.body);
});
✅ Solución: 
const rateLimit = require('express-rate-limit');
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 5 // 5 requests per 15 minutes
});
app.post('/login', limiter, (req, res) => {...});

Severidad: MEDIO

CVSS Score: 5.3
